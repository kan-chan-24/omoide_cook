# README
## アプリタイトル：あのとき食べたあれ
自身の料理レシピをエピソードと一緒に投稿・閲覧ができるアプリです。

## デプロイ
- Renderで公開予定
- URL: ***

## サービス概要
このアプリは、「料理にまつわるエピソードを投稿する」サービスを提供します。
風邪を引いた時に母が出してくれた看病飯の思い出、貧乏時代のささやかな贅沢飯の記憶など、
**自分を支えてくれた料理の思い出にレシピを添え、ユーザーは自由に投稿・閲覧**することができます。

## このサービスへの思い・きっかけ
私の家庭の食卓では、夫婦それぞれが作った料理に関連した話題で盛り上がる事がよくあります。
「風邪を引いた時に母が出してくれたコンソメスープにだけ胡椒がたっぷり入っていた」
「粘りが強めのカレーを食べると、おばあちゃんがパパッと出してくれるカレーを思い出す」
そんな会話をしていると、じゃあ次はこんな料理を作ろう。あんな食材を使ってみよう。と次の献立決めがワクワクする体験になったりします。

時短料理やコスパ献立など、パフォーマンスの良い献立が好まれるこの頃ですが、
「レシピサイトではなく、**読み物としてレシピを書いた人の思い入れやエピソードの方を楽しめるコンテンツがあっても面白いかも**」と思ったことが
本サービスを思いついたきっかけです。

振り返ってみると自分自身、献立を選ぶ時に、効率や在庫消費を考えて、料理を楽しめていない日もあったかと思います。
もちろん、それは家庭の食品管理のために、本当に大切なことです。
ただ、たまには完成した味に思いを馳せ、調理工程に趣向を凝らし、料理を楽しむための手助けがあってもいいのかもしれません。
**「料理を『作業』ではなく『体験』として楽しむための、物語とレシピのプラットフォーム」**が本アプリの目指す形です。

## このアプリで解決したい課題
**「レシピとの偶然の出会い」が少ないこと**
- 既存のレシピサイトは 「検索ベース」 なので、自分が知っている範囲の食材や料理名で探すことが多い。
- そのため、知っている料理、マンネリ化、同じレシピばかり、知らなかった素敵なレシピとは出会いにくい。

💡**解決案**
- このアプリなら「エピソードを読む → 「作ってみたい！」と思う → レシピに出会う」
- 「食材検索」ではなく「物語から始まるレシピとの出会い」 という新しい体験を提供できる。

**料理が「義務感」や「作業」になっていること**
- 料理を作る「わくわく」よりも在庫消費をやり切るための「義務感」を感じている
- 冷蔵庫の在庫を消費するための献立選び、時短レシピの追求。これらは効率化には大切だけど、 料理の楽しさや温かさは失われがちなのかも。

💡**解決案**
- エピソードを読むことで「作ってみたい」という**感情が動く**
- 料理が「作業」から「誰かの思い出を追体験する体験」に変わる
- **「効率」ではなく「体験」を提供する**

**「料理の背景にある物語」を知る場がない**
- 当たり前だけど**既存のレシピサイトのメインは 「作り方」。** 
- でも、人は物語や背景に心を動かされる。

💡**解決案**
- レシピだけでなく**「なぜこの料理を作るのか」「どんな思い出があるのか」**が共有される
- 読み物として楽しめる
- **「レシピサイト」ではなく「料理エッセイ×レシピ」のプラットフォーム**

## ターゲットユーザー
メイン層：読み物（料理エッセイ）と合わせて料理を楽しみたい人
年齢　　：30代〜50代
状況　　：料理エッセイや食に関する物語が好き
悩み　　：レシピサイトは味気ない、でも料理本は重い
提供できる価値：短いエピソードを読みながら、実際に作れるレシピも得られる

サブ層：料理を楽しみたい人
年齢　：20代〜30代
状況　：料理は好きだけど、最近は義務感で作っている
悩み　：「作りたい」と思える動機が欲しい
提供できる価値：物語を読んで心が動き、料理が楽しくなる

サブ層：マンネリ化している料理好き
年齢　：30代〜40代
状況　：毎日料理をしているが、同じレシピばかりで飽きている
悩み　：新しいレシピに出会いたいけど、検索しても同じようなものばかり
提供できる価値：エピソードを通じて、知らなかったレシピに偶然出会える

#### 「なぜ読み物と合わせて料理を楽しみたい人」をメインターゲットにしたか
- **明確な「既存サービスにない価値」を提供できる**
    - レシピサイトは味気ない、でも料理エッセイはどう探せばいいのかわからない
    - この悩みはレシピサイトでは解決できていない領域かもしれない
        - **クックパッド等のレシピサイト** → 効率的だが味気ない
        - **料理本・料理エッセイ** → 物語性はあるが重い、すぐに作れない
        - **このアプリ** → 短いエピソードを読みながら、実際に作れるレシピも得られる
    
    **「レシピサイト」と「料理エッセイ」の間を埋める、新しいポジション** を取れる
- **「読み物」という入り口が強い**
    - このターゲットは **「料理を作るため」ではなく「読み物として楽しむため」にアプリを開く** 可能性が高い
    - ユーザーが **能動的に** アプリを開く動機がある
    - 「今日の献立どうしよう」という切迫した状況でなくても使える
    - **隙間時間に読む → 心が動く → 後で作ってみる** という流れが生まれる
    
    **「検索ツール」ではなく「読み物アプリ」として習慣化しやすい**
    
- **投稿のハードルが低い**
    - このターゲット層は **「料理エッセイや食に関する物語が好き」**
    - 自分もエピソードを書いてみたいと思いやすい
    - 投稿の質が高くなりやすい（物語を書くことに抵抗がない）
    - **コミュニティが育ちやすい**

## 既存サービスとの比較
#### **既存レシピサイト（クックパッド、デリッシュキッチンなど）**

- **目的**: 効率的にレシピを検索して作る
- **検索方法**: 食材名、料理名で検索
- **体験**: 「作り方を知る」

#### 本サービス

- **目的**: 物語を楽しみながら、作ってみたいレシピに出会う
- **検索方法**: エピソードを探す、感情や思い出のタグで探す
- **体験**: 「心が動いてから作りたくなる」

あくまで**「レシピ検索サイト」ではなく「料理エッセイ×レシピのプラットフォーム」**

## できること（機能一覧）
**Must（MVPリリース）**
- ユーザー登録・ログイン
- エピソード＋レシピの投稿機能
- 投稿一覧の表示（タイムライン形式）
- 投稿詳細の表示
    - 投稿の編集
    - 投稿の削除
- 画像添付

**Should（優先的に実装）**
- いいね機能
- コメント機能
- タグやカテゴリ機能
- 検索機能（エピソードにつけられたタグ、食材、料理名）
- ランダム表示機能（”今日の献立”のようなイメージ）

**Could（後から追加実装）**
- フォロー機能
- 公開/非公開設定
- マイプロフィール機能
- レシピ保存機能
- エピソード作成のAIサポート機能
